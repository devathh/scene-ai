# Prompt for AI Scenario Generation

## Task
Generate a valid JSON object representing a **Scenario** entity based on the user's request. 

## Strict Requirements
1. **Output Format**: Return **ONLY** a raw JSON object. Do not include markdown code blocks (```json), explanations, or any text outside the JSON structure.
2. **Schema Compliance**: The JSON must strictly match the following structure intended for the `scenario.New` constructor.
3. **No Scenes**: Do **NOT** generate the `scenes` array. The output must contain only the high-level scenario metadata.
4. **Data Types**: Ensure strict adherence to data types (strings for text, integers for status).

## Target JSON Structure
```json
{
  "title": "string (required, non-empty)",
  "global_style_prompt": "string (required, non-empty, visual style description)",
}
```

## Field Definitions
- `title`: A concise, catchy title for the scenario.
- `global_style_prompt`: A description of the visual style, camera angles, lighting, and mood applicable to the whole video.

## Example Input
"Create a cyberpunk detective story set in Neo-Tokyo."

## Example Output
{"title":"Neon Shadows","global_style_prompt":"Cyberpunk aesthetic, neon lights, heavy rain, dark moody atmosphere, high contrast, cinematic lighting."}

## Negative Constraints
- DO NOT output the `scenes` field.
- DO NOT output `id`, `author_id`, `created_at`, or `updated_at` (these are generated by the backend).
- DO NOT wrap the output in markdown blocks.
- DO NOT add comments or extra whitespace.
